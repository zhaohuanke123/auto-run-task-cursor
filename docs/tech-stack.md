## 2. Unity 核心技术栈

### 2.1 UGUI 系统

**核心组件**：

| 组件 | 用途 | 学习要点 |
|------|------|---------|
| Canvas | UI容器 | 渲染模式、层级管理 |
| RectTransform | 布局 | 锚点、锚框、布局系统 |
| EventSystem | 事件 | 点击、拖拽、自定义事件 |
| Mask | 裁剪 | 遮罩、裁剪优化 |

**进阶技能**：

```
UGUI 进阶路径
│
├── 基础应用
│   ├── 组件使用
│   ├── 布局系统
│   └── 事件系统
│
├── 性能优化
│   ├── DrawCall 合批
│   ├── 图集管理
│   ├── UI 重绘优化
│   └── 对象池复用
│
└── 架构设计
    ├── UI 框架设计
    ├── MVVM 模式应用
    └── 数据绑定系统
```

**性能优化要点**：

| 问题 | 解决方案 |
|------|---------|
| DrawCall 过多 | 图集合并、动静分离 |
| UI 重绘频繁 | 脏标记、分 Canvas |
| 内存占用大 | 对象池、资源卸载 |
| 响应慢 | 分帧加载、异步加载 |

### 2.2 URP（通用渲染管线）

**URP 架构**：

```
URP 渲染流程
│
├── Render Features（自定义渲染特性）
│   ├── 渲染顺序控制
│   ├── 后处理效果
│   └── 自定义 Pass
│
├── Renderer Data（渲染器配置）
│   ├── 层级剔除
│   ├── 阴影设置
│   └── 后处理栈
│
└── URP Asset（全局设置）
    ├── 质量等级
    ├── 渲染设置
    └── 光照设置
```

**学习要点**：

| 阶段 | 内容 | 时间 |
|------|------|------|
| 入门 | URP 基础配置、Shader Graph | 2周 |
| 进阶 | 自定义 Render Feature | 2周 |
| 高级 | 手写 Shader、渲染管线定制 | 4周+ |

### 2.3 DOTS（数据导向技术栈）

**DOTS 组成**：

| 组件 | 说明 | 状态 |
|------|------|------|
| ECS | 实体组件系统 | 核心稳定 |
| Burst Compiler | 高性能编译器 | 生产可用 |
| Job System | 多线程任务系统 | 生产可用 |
| Collections | 安全集合 | 生产可用 |

**ECS 架构示意**：

```csharp
// 传统 OOP 方式
public class Enemy : MonoBehaviour {
    public float health;
    public float speed;
    void Update() { /* 逻辑 */ }
}

// ECS 方式
public struct EnemyData : IComponentData {
    public float health;
    public float speed;
}

public class EnemyMovementSystem : SystemBase {
    protected override void OnUpdate() {
        Entities.ForEach((ref EnemyData enemy) => {
            // 高性能并行处理
        }).ScheduleParallel();
    }
}
```

**适用场景**：

- 大量同类型实体（如敌人、子弹、粒子）
- 需要高性能计算
- 可并行处理的逻辑

### 3.2 渲染管线知识

**渲染管线阶段**：

```
应用阶段(App) ──→ 几何阶段 ──→ 光栅化阶段 ──→ 像素处理
     │              │              │              │
   场景数据       顶点变换       三角形裁剪      纹理采样
   剔除          投影          片元生成        颜色混合
   LOD           裁剪                         深度测试
```

**关键概念**：

| 概念 | 说明 | 应用 |
|------|------|------|

| Z-Test | 深度测试 | 控制渲染顺序 |
| Blend | 混合 | 透明效果 |
| Cull | 剔除 | 背面剔除 |

### 3.3 GPU 编程入门

**HLSL 基础结构**：

```hlsl
Shader "Custom/SimpleShader" {
    Properties {
        _MainTex ("Texture", 2D) = "white" {}
        _Color ("Color", Color) = (1,1,1,1)
    }
    
    SubShader {
        Pass {
            HLSLPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            
            struct Attributes {
                float4 positionOS : POSITION;
                float2 uv : TEXCOORD0;
            };
            
            struct Varyings {
                float4 positionCS : SV_POSITION;
                float2 uv : TEXCOORD0;
            };
            
            Varyings vert(Attributes input) {
                Varyings output;
                output.positionCS = TransformObjectToHClip(input.positionOS.xyz);
                output.uv = input.uv;
                return output;
            }
            
            half4 frag(Varyings input) : SV_Target {
                return _Color;
            }
            ENDHLSL
        }
    }
}
```

## 4. .NET/CLR 深度知识

### 4.1 CLR 核心概念

**运行时架构**：

```
┌─────────────────────────────────────────┐
│                 C# 代码                  │
├─────────────────────────────────────────┤
│              编译器 (Roslyn)             │
├─────────────────────────────────────────┤
│                IL 代码                   │

│  ┌─────────────────────────────────┐    │
│  │ JIT 编译器 │ GC │ 类型系统 │ 安全│    │
│  └─────────────────────────────────┘    │
├─────────────────────────────────────────┤
│              机器码执行                  │
└─────────────────────────────────────────┘
```

### 4.2 垃圾回收（GC）机制

**Unity GC 特点**：

| 特点 | 说明 | 影响 |
|------|------|------|
| 增量式 GC | 分帧执行 GC | 减少卡顿 |
| Boehm GC | 非分代、非压缩 | 容易内存碎片 |
| 手动触发 | GC.Collect() | 可控但需谨慎 |

**GC 优化策略**：

```
GC 优化原则
│
├── 减少分配
│   ├── 对象池
│   ├── 预分配缓冲区
│   └── 避免临时字符串
│
├── 避免装箱
│   ├── 使用泛型
│   └── 避免值类型转 object
│
└── 控制触发时机
    ├── 手动触发
    └── 分帧加载
```

### 4.3 内存管理最佳实践

**内存分配优化**：

| 场景 | 优化方法 |
|------|---------|
| 频繁创建对象 | 对象池 |
| 字符串拼接 | StringBuilder |
| 数组操作 | 预分配、避免 Resize |
| 协程 | 避免每帧 yield return new |

**Unity 内存分析工具**：

- Memory Profiler：内存快照分析
- Profiler：运行时内存监控
- Asset Auditing：资源审计

## 6. 性能优化技术

### 6.1 性能分析工具

| 工具 | 用途 | 使用场景 |
|------|------|---------|
| Profiler | 综合性能分析 | 定位瓶颈 |
| Frame Debugger | 渲染分析 | DrawCall 问题 |
| Memory Profiler | 内存分析 | 内存泄漏 |
| RenderDoc | 帧捕获分析 | 渲染问题 |

### 6.2 CPU 优化

**常见问题与解决方案**：

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| GC 过多 | 频繁内存分配 | 对象池、预分配 |
| 物理计算慢 | 复杂碰撞 | 简化碰撞体 |
| 脚本执行慢 | 算法效率 | 优化算法、缓存 |
| 协程过多 | 每帧检查 | 合并、优化 |

**优化技巧**：

```csharp
// 优化前：每帧 GetComponent
void Update() {
    var renderer = GetComponent<Renderer>();
}

// 优化后：缓存引用
private Renderer _renderer;
void Awake() {
    _renderer = GetComponent<Renderer>();
}
```

### 6.3 GPU 优化

**渲染优化策略**：

| 策略 | 说明 | 效果 |
|------|------|------|
| 合批 | 合并相同材质的网格 | 减少 DrawCall |
| LOD | 根据距离切换模型 | 减少渲染负担 |
| 遮挡剔除 | 剔除不可见物体 | 减少渲染量 |
| GPU Instancing | 相同网格一次绘制 | 大幅减少 DrawCall |

**Shader 优化**：

```hlsl
// 优化前：分支过多
if (condition1) { ... }
else if (condition2) { ... }

// 优化后：使用数学运算
result = lerp(value1, value2, condition);
```

### 6.4 内存优化

**内存类型**：

| 类型 | 说明 | 优化方向 |
|------|------|---------|
| Native Memory | 引擎原生内存 | 资源压缩 |
| Managed Memory | C# 托管堆 | 减少 GC |
| Graphics Memory | 显存 | 纹理压缩 |

**资源优化**：

| 资源类型 | 优化方法 |
|---------|---------|
| 纹理 | 压缩格式、Mipmap、图集 |
| 模型 | 减面、LOD、骨骼简化 |
| 音频 | 压缩格式、流式加载 |
| 动画 | 动画压缩、骨骼简化 |

## 7. 跨平台与热更新方案

### 7.1 热更新技术对比

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| Lua | 脚本语言 | 成熟稳定 | 性能差、维护成本高 |
| ILRuntime | IL 解释执行 | 纯 C# | 性能一般、调试困难 |
| HybridCLR | 原生 AOT + 解释 | 性能好、体验好 | 需要处理 AOT 泛型 |

### 7.2 HybridCLR 方案

**工作原理**：

```
C# 代码 ──→ IL 代码 ──→ AOT 编译 ──→ 原生代码
                          │
                          └── 解释执行部分（热更新代码）
```

**集成步骤**：

1. 安装 HybridCLR 包
2. 配置热更新程序集
3. 处理 AOT 泛型
4. 打包与加载

### 7.3 跨平台发布

**支持平台**：

| 平台 | 特点 | 注意事项 |
|------|------|---------|
| iOS | App Store 审核 | 代码合规、隐私 |
| Android | 多渠道、碎片化 | 适配机型、包体优化 |
| PC | 性能要求高 | 多分辨率、输入适配 |
| WebGL | 即点即玩 | 性能受限、兼容性 |
| 小游戏 | 轻量化 | 包体限制、能力限制 |
