## 3. Prompt 库与最佳实践

### 3.1 Prompt 设计原则

**核心原则**：

1. **明确目标**：清楚说明要做什么
2. **提供上下文**：给出必要的背景信息
3. **指定约束**：说明技术栈、规范要求
4. **定义输出**：说明期望的输出格式

**Prompt 结构模板**：

```
[角色定义] 你是一个 Unity 游戏开发者

[上下文] 我正在开发一个 XXX 功能，使用 Unity 2021 和 C#

[任务] 请帮我实现 YYY 功能

[约束] 
- 使用 Unity 最佳实践
- 代码要有注释
- 考虑性能优化

[输出] 提供完整代码和简要说明
```

### 3.2 常用 Prompt 模板库

**代码生成类**：

```markdown
# 功能实现
"请帮我实现一个 Unity Editor 工具窗口，包含：
1. 搜索框：用于输入搜索关键词
2. 结果列表：显示搜索结果
3. 快捷操作按钮

要求：
- 使用 EditorWindow
- 支持 Unity 2021+
- 代码有中文注释"

# 代码优化
"请优化以下代码，重点关注：
1. 性能优化（减少 GC）
2. 代码可读性
3. 异常处理

[粘贴代码]"

# Bug 修复
"以下代码出现错误：[错误信息]
代码：
[粘贴代码]

请分析原因并提供修复方案。"
```

**学习研究类**：

```markdown
# 概念解释
"请用简单易懂的方式解释 Unity 中的 [概念]，
包括：
1. 是什么
2. 为什么需要
3. 怎么使用
4. 常见问题"

# 技术对比
"请对比 Unity 中的 [技术A] 和 [技术B]：
1. 核心区别
2. 各自优缺点
3. 适用场景
4. 选择建议"

# 学习路径
"我想学习 [技术领域]，请给出：
1. 学习路径（从基础到进阶）
2. 推荐资源
3. 实践项目建议"
```

**文档生成类**：

```markdown
# 代码文档
"请为以下代码生成文档：
1. 功能说明
2. 参数说明
3. 返回值说明
4. 使用示例

[粘贴代码]"

# API 文档
"请为这个类生成 API 文档，格式参考 Unity 官方文档风格：

[粘贴类代码]"
```

### 3.3 Prompt 优化技巧

**迭代优化法**：

```
第一轮 Prompt：
"帮我写一个对象池"

第二轮（优化）：
"帮我写一个 Unity 通用的对象池，要求：
- 支持泛型
- 可以设置初始容量和最大容量
- 支持预热
- 使用 C# 实现"

第三轮（进一步优化）：
"请实现一个高性能的 Unity 对象池系统：
- 使用泛型支持任意类型
- 实现对象生命周期管理
- 支持异步预热
- 提供统计和调试功能
- 使用 [约束条件]..."
```

## 6. AI 辅助调试流程

### 6.1 调试场景分类

| 场景 | AI 辅助方式 | 效果 |
|------|-----------|------|
| 编译错误 | 直接粘贴错误信息 | 极高 |
| 运行时错误 | 提供错误+代码 | 高 |
| 逻辑错误 | 描述预期+实际 | 中 |
| 性能问题 | 提供 Profiler 数据 | 高 |

### 6.2 调试 Prompt 模板

```markdown
# 编译错误
"以下代码编译报错，请分析原因并修复：
错误信息：
[粘贴错误信息]

代码：
[粘贴代码]"

# 运行时错误
"代码运行时出现以下错误，请帮我分析和修复：
错误信息：
[粘贴错误信息]

堆栈跟踪：
[粘贴堆栈]

相关代码：
[粘贴代码]"

# 逻辑问题
"代码没有报错，但结果不符合预期：

预期行为：[描述预期]
实际行为：[描述实际]

代码：
[粘贴代码]

请帮我找出问题所在。"

# 性能问题
"以下代码存在性能问题：
- 现象：[描述问题]
- Profiler 数据：[如有]

代码：
[粘贴代码]

请分析原因并提供优化方案。"
```

### 6.3 调试工作流

```
AI 辅助调试流程

发现问题
    │
    ▼
┌─────────────┐
│ 收集信息    │
│ - 错误信息  │
│ - 代码上下文│
│ - 预期行为  │
└─────┬───────┘
      │
      ▼
┌─────────────┐
│ AI 分析     │
│ (Cursor Chat)│
└─────┬───────┘
      │
      ├──▶ 找到原因 ──▶ 修复
      │
      └──▶ 需要更多信息 ──▶ 补充信息
```
