### 2.3 GPU 架构

**GPU 与 CPU 对比**：

| 特性 | CPU | GPU |
|------|-----|-----|
| 核心数 | 少（几个到几十） | 多（数千） |
| 单核性能 | 强 | 弱 |
| 并行能力 | 有限 | 极强 |
| 适用场景 | 复杂逻辑 | 大量相同运算 |
| 内存带宽 | 一般 | 高 |

**GPU 渲染流程优化**：

```
GPU 优化要点
│

│   ├── 变换优化
│   └── 裁剪效率
│
├── 光栅化
│   ├── 三角形设置

│
├── 像素处理

│   └── Early-Z
│
└── 输出合并
    ├── 颜色混合
```

## 3. 主流渲染技术

### 3.1 PBR（基于物理的渲染）

**PBR 核心概念**：

| 概念 | 说明 |
|------|------|
| 微表面模型 | 表面由微小镜面组成 |
| 能量守恒 | 出射光不超过入射光 |
| 菲涅尔效应 | 角度影响反射强度 |
| 金属度 | 区分金属与非金属 |

**PBR 参数**：

| 参数 | 作用 | 范围 |
|------|------|------|
| Albedo | 基础颜色 | RGB 颜色 |
| Metallic | 金属度 | 0-1 |
| Roughness | 粗糙度 | 0-1 |
| Normal | 法线贴图 | RGB 法线 |
| AO | 环境光遮蔽 | 0-1 |

### 3.2 光照技术

**光照模型演进**：

```
Lambert（漫反射）
    │
    └── Blinn-Phong（高光）
            │
            └── PBR（基于物理）
                    │
                    └── 全局光照
                            │
                            ├── 烘焙 GI
                            ├── 实时 GI
                            └── 光线追踪
```

**阴影技术**：

| 技术 | 原理 | 适用场景 |
|------|------|---------|
| Shadow Map | 深度比较 | 通用 |
| Cascaded Shadows | 级联阴影 | 大场景 |
| VSM | 方差阴影 | 柔和阴影 |
| PCF | 百分比滤波 | 柔和边缘 |

### 3.3 后处理效果

**常用后处理**：

| 效果 | 作用 | 实现复杂度 |
|------|------|-----------|
| Bloom | 泛光 | 低 |
| Tone Mapping | 色调映射 | 低 |
| Color Grading | 调色 | 低 |
| Depth of Field | 景深 | 中 |
| Motion Blur | 运动模糊 | 中 |
| SSAO | 屏幕空间环境光遮蔽 | 中 |
| SSR | 屏幕空间反射 | 高 |

### 3.4 全局光照

**GI 技术对比**：

| 技术 | 类型 | 效果 | 性能 |
|------|------|------|------|
| Lightmaps | 烘焙 | 静态场景 | 低运行成本 |
| Light Probes | 烘焙 | 动态物体 | 低 |
| Realtime GI | 实时 | 动态光照 | 高 |
| SSGI | 屏幕空间 | 局部效果 | 中高 |
| Ray Tracing | 实时 | 最真实 | 最高 |

## 4. Shader 开发学习路径

### 4.1 Shader 语言对比

| 语言 | 平台 | 特点 |
|------|------|------|
| HLSL | DirectX/Windows | Unity URP 主要使用 |
| GLSL | OpenGL/跨平台 | 标准跨平台 |
| ShaderLab | Unity | Unity 封装层 |
| CG | NVIDIA | 已逐渐淘汰 |

### 4.2 学习阶段规划

**第一阶段：Shader Graph（2-4周）**

```
学习内容
│
├── 节点基础
│   ├── 输入节点
│   ├── 数学节点
│   ├── 纹理节点
│   └── 输出节点
│
├── 常用效果
│   ├── 基础材质
│   ├── 流动效果
│   ├── 溶解效果
│   └── 菲涅尔效果
│
└── 练习项目
    ├── 卡通着色
    ├── 水面效果
    └── 全息效果
```

**第二阶段：ShaderLab 基础（4-8周）**

```hlsl
// ShaderLab 基础结构
Shader "Custom/BasicShader" {
    Properties {
        _MainTex ("Texture", 2D) = "white" {}
        _Color ("Color", Color) = (1,1,1,1)
    }
    
    SubShader {
        Tags { "RenderType"="Opaque" }
        LOD 100
        
        Pass {
            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            
            // 顶点着色器
            // 片元着色器
            
            ENDCG
        }
    }
}
```

**第三阶段：HLSL 深入（8-16周）**

```
深入学习内容
│
├── URP Shader 编写
│   ├── URP 结构理解
│   ├── HLSL 语法
│   └── 自定义 Pass
│
├── 高级效果
│   ├── PBR 扩展
│   ├── 自定义光照
│   └── 后处理 Shader
│
└── 优化技巧
    ├── 指令优化
    ├── 分支优化
    └── 精度控制
```

### 4.3 Shader 开发最佳实践

**性能优化原则**：

| 原则 | 说明 |
|------|------|
| 减少计算 | 将计算移到顶点着色器 |
| 避免分支 | 使用数学替代 if-else |
| 精度控制 | 适当使用 half/fixed |
| 纹理采样 | 减少、合并采样 |
| 变体管理 | 减少编译变体数量 |

## 6. 业界专家成长经验

### 6.1 典型成长路径

**路径一：学术路线**

```
计算机/数学专业 → 图形学研究 → 论文发表 → 工业界应用
     │                │
     └── 基础扎实      └── 深度理论
```

**路径二：工程路线**

```
游戏开发 → 自学图形学 → 小项目实践 → 大项目经验 → 专家
     │            │
     └── 实践经验  └── 解决实际问题
```

### 6.2 专家建议汇总

| 建议 | 说明 |
|------|------|
| 数学是基础 | 线性代数、微积分必须扎实 |
| 实践驱动 | 通过项目学习，而非只看书 |
| 源码阅读 | 阅读开源引擎和Shader源码 |
| 关注前沿 | 跟踪 SIGGRAPH、GDC 等会议 |
| 写作分享 | 通过输出加深理解 |

### 6.3 常见误区

| 误区 | 正确认识 |
|------|---------|
| 数学太难 | 循序渐进，边用边学 |
| 必须读论文 | 工程实现更重要 |
| 只学Unity | 了解底层原理更关键 |
| 效果越炫越好 | 性能与效果平衡 |

```
├── ShaderLab 语法

第7-12个月：深入进阶
├── PBR 原理与实现
├── 后处理效果
├── 性能优化
└── 完成综合渲染效果

├── 选择细分领域
```
